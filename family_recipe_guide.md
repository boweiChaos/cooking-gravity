# 家庭专属菜谱小程序开发指南 (Cooking Gravity)

> 💡 **项目愿景**：为老婆量身定制的极轻量级家庭应用。主打“极客式浪漫”——周末早晨提需求，下午即可上线体验。核心诉求为记录日常菜单、选菜，仅供双人使用。

---

## 一、 架构选型：为什么是【微信云开发 (TCB)】？

对于此类“玩具级/自用级”双人微型应用，传统的“客户端 -> Golang 接口 -> MySQL 数据库”架构过于沉重且开发成本极高。

**微信云开发架构的四大碾压级优势：**
1. **零服务器与运维成本**：免去了购买服务器、配置 Nginx、申请域名和繁琐的 HTTPS/ICP 备案审核。
2. **真正的“零后端”体验**：前端基于 JS/TS 直接操作云数据库（类似 MongoDB），无需编写任何后台 API 接口。
3. **天生自带登录鉴权**：系统从底层自动记录访问者的微信唯一身份标识（`openid`），免去编写 `wx.login` 和 Token 逻辑。
4. **无需上架审批**：直接生成“体验版”二维码给家人扫码即可使用，彻底摆脱官方冗长的审核流程。

## 二、 核心环境配置与“避坑”实录

### 1. 物理环境隔离法则
**铁律**：绝不能复用商业主项目（如 `Persona`）的开发环境！
* **操作**：使用独立邮箱注册并申请**全新的小程序 AppID**。
* **开发工具建项**：后端服务选项**必须勾选【微信云开发】**（推荐选用“云开发基础模板”，自带初始化代码脚手架）。

### 2. 免费额度激活与资费避坑
在开发者工具的“云开发”控制台中开通服务时，遭遇并跨越了“微信悄悄下线免费版”的陷阱：
* **避坑 1：必须关闭超额扣费**。务必关闭“使用超出配额部分按量付费”的开关。作为 2 人应用，防范由于前端逻辑死循环导致的恶意扣费（加上“物理熔断器”）。
* **避坑 2：拒绝自动包月**。如果无法找到免费试用，也应选择“单月 ¥19.9”。
* **终极白嫖成功**：经过控制台页面的反复探索与点击，成功触发并领取了隐藏的 **“免费开发环境”** 试用期套餐。

## 三、 极简全栈实施流程

### 第一步：代码级数据库连接 (SDK 初始化)
在云开发控制台中获取你的 **唯一环境 ID**（例如图中的 `cloud1-0g65hyz8b91894f3`）。在小程序的全局文件 `app.js` 或 `app.ts` 的启动生命周期中，插入钥匙：

```javascript
App({
  onLaunch() {
    if (!wx.cloud) {
      console.error('请使用 2.2.3 或以上的基础库以使用云能力')
    } else {
      wx.cloud.init({
        env: 'cloud1-0g65hyz8b91894f3', // 将云开发控制台内的 环境ID 填入此处
        traceUser: true, // 核心机制：自动在每条落库数据上打上用户的 openid
      })
    }
  }
})
```

### 第二步：云控制台可视化工单 (建表)
* 点击云开发控制台的“数据库”面板。
* 点击“添加集合”（即关系型数据库中的 Table），输入名称 `recipes`。

### 第三步：前端“直插”云端 (发功写数据)
在任何业务页面（如“新增菜谱”按钮的点击事件中），写入以下极简 JS 代码，直接完成前后端数据交互闭环：

```javascript
// 在 pages/add_recipe/index.js 中
const db = wx.cloud.database()

function addRecipe() {
  db.collection('recipes').add({
    data: {
      name: '红烧肉',       // 菜名
      cook: '妻子',         // 标签
      date: new Date()      // 记录时间
    },
    success: function(res) {
      // 成功后，查看数据库会发现数据自动带上了 _openid 字段
      console.log("云端写入成功啦！云端唯一数据 ID：", res._id)
    },
    fail: console.error
  })
}
```
